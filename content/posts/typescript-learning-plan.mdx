TypeScript is easiest to learn when it is tied to real code. My plan is to move from syntax to habits, then from habits to design.

## Stage 1: Everyday types

Focus on the types that appear constantly:

- `string`, `number`, `boolean`
- arrays and objects
- union types
- optional fields
- function parameters and return values

## Stage 2: Component props

React components are a good place to practice types because props define a small contract. Start with explicit prop types, then extract shared types only when reuse is real.

## Stage 3: Data modeling

The next step is modeling data from files, APIs, and forms. The goal is to make invalid states harder to represent.

## Practice rule

When a type becomes confusing, write a tiny example first. Small examples reveal whether the problem is TypeScript or unclear data design.
